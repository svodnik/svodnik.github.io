Below you will find homework assignments; these are assigned on Thursday and due the following Tuesday. This page will be updated weekly.

WEEK 01

Homework: Due Tuesday, 3/14

  • Finish your basic portfolio site:
    1. Download starter code here
    2. Build two pages - an 'About Page' and a 'Portfolio' - that link to one another.
    3. Link both pages together using an anchor tag
    4. Use a unique title and a single unique h1 tag
    5. Show images using img tags including unique alt attributes
    6. Use an external CSS stylesheet to style your pages
    7. Select at least 5 elements on the page and practice using the CSS properties we've covered in class (you should use at least 6 of the properties we've covered in your CSS file)
    8. Use at least 6 different HTML tags (h1, h2, h3, h4, h5, h6, p, ul, ol, li, a, img, etc.) on your pages
    9. Follow naming conventions, maintain consistency across your .html and .css files
    10. Indent nested elements to increase your code's readability

WEEK 06

Homework: Due Tuesday, 4/18

  1. Rock, Paper, Scissors (started in class)

    Technical Requirements:
    • When the user clicks a button, the user's move and the computer's move should be printed out to the page (e.g. "You chose rock. The computer chose paper.").
    • The winner of that round should be displayed (e.g. "You win! :)").
    • Each "round", the winner's score should be incremented by 1 (unless it is a tie).
    • The updated score should be displayed on the page.
    • Code should use proper indentation and should be free of syntax errors. You can use JSHint to check for errors.
    • Code should avoid repetition (DRY!) by using one or more functions.
    • Bonus: Create a "best of three" version. After three points have been scored, show some sort of visual feedback to let the user know who the final winner is. Get creative!
    • Bonus (worth 4 extra points): Refactor the Temp Converter lab from the previous class to avoid repetition by using a function.

  2. JavaScript pseudocode for final project

    Think about user interactions, animations, forms, and other parts of your final project that will use JavaScript, and outline this behavior using pseudocode.

    Don't worry about making your pseudocode too detailed (unless you have a very clear idea of what you need your code to do and how you will structure it).

    At a minimum, your pseudocode should describe what data you need to get from users, what data you need to establish in your code, and what steps you will take using the data, and what event(s) will cause your code to run.

  3. Start working on HTML/CSS for final project

    Plan to spend an hour or two making progress. Write your code in the template provided in the starter code.
